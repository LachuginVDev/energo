# Energo — лендинг

Одностраничный лендинг на чистом HTML/CSS/JS без фреймворка. Сборка через Vite; разметка в основном в `index.html`, стили и скрипты подключаются из `src/`.

## Стек

- **Vite** — dev-сервер и production-сборка в `dist/`
- **Swiper** — слайдеры (услуги, объекты, типы работ, партнёры, отзывы, доверие)
- Шрифты **Geist** и **Nunito Sans** (Google Fonts, подключение в `<head>`)

## Команды

| Команда | Назначение |
|--------|------------|
| `npm run dev` | Разработка, порт `5173` |
| `npm run build` | Сборка в `dist/` |
| `npm run preview` | Просмотр собранного `dist/` |
| `npm run build:portable` | Переносимая сборка (скрипт `tools/build-portable.mjs`) |

## Верстка и структура

### Разметка

- Точка входа: **`index.html`** — все секции лендинга внутри `<main class="landing">`.
- Семантика: `<section>` с якорями `id`, заголовки по смыслу, где нужно — `aria-label`.
- Контентные картинки и видео через **`<img>`** и **`<video>` / `<source>`** с путями в `./src/assets/...` — Vite обрабатывает их при сборке; так удобнее переносить блоки (в т.ч. в WordPress), чем фоновые картинки только из CSS.

### Стили

- **`src/styles/global.css`** — сброс/база, `.container`, кнопки `.btn`, общий паттерн **`.section-heading`** (мини-лейбл + крупный лид), CSS-переменные (`:root`): палитра, `--page-pad`, **`--section-pad-top` / `--section-pad-bottom`** для вертикальных ритмов секций, семейства шрифтов.
- **`src/blocks/<блок>/<блок>.css`** — стили секции рядом с её JS (hero, about, services, …).
- Именование близко к **БЭМ**: блок `hero`, элементы `hero__content`, модификаторы `hero__row--1`.
- **Адаптив**: комбинация `@media` и **`clamp()`** для типографики и отступов, чтобы между десктопом и мобилой не было «ступенек» только из-за фиксированных `px`.

### Скрипты

- **`src/main.js`** — импорт `global.css` и модулей блоков по порядку появления на странице.
- **`src/blocks/<блок>/<блок>.js`** — инициализация слайдеров, открытие модалок, мелкая интерактивность; без общего SPA-роутера.

### Сборка и пути

В `vite.config.js` задано **`base: './'`** — относительные пути к ассетам в `dist/`, что упрощает выкладку в подпапку на сервере или перенос в CMS.

## Каталоги (кратко)

```
index.html          — разметка страницы
src/
  main.js           — вход фронта
  styles/global.css — глобальные стили и переменные
  blocks/           — секции: css + js парами
  assets/images|videos
tools/build-portable.mjs
```

## Требования

Node.js с npm (версии с поддержкой ES modules, как в `package.json`).
